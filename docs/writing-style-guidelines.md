# Writing Style Guidelines（內容寫作風格規範）

目標：把內容寫得更像「自然敘述＋可決策的旅行筆記」，主要以網路搜集到的事實/資料為底做潤飾與整理；避免省力語氣硬導向，也不靠到處畫重點。（真實性底線見下方「真實性與來源」。）

適用範圍（SSOT）：

- `docs/src/content/attractions.*.md`
- `docs/src/content/stays.*.md`
- `docs/src/content/transport.*.md`
- `docs/src/content/extensions.*.md`
- `docs/src/content/itinerary.md`
- `docs/src/content/misc.md`
- `docs/src/content/README.md`

搭配規範：

- 粗體使用請遵守：`docs/bold-slimming-guidelines.md`

## 核心寫作原則（最重要的 7 條）

- 短文優於列點（先段落，後清單）：短文正文一律先用 1–2 段（每段 2–5 句）把「這是什麼／為什麼值得」講完；清單只拿來放可比較的決策欄位。
- 不捏造（SSOT 底線）：不是你真的經歷過的就不要寫成第一人稱；不從照片推論味道/服務/人潮。
- 資料不足先查再寫（可執行）：缺交通/時間/票價/規則/限制條件就上網補齊，並對關鍵資訊做交叉驗證；來源互相打架就不要寫死。
- 氛圍用可觀測線索寫：把氛圍落在光線/材質/動線/視野等線索；必要時用圖片搜尋補「看得到」的觀察點。
- 先給脈絡再給建議（選項語氣）：先說「這是什麼／為什麼值得」，再說「怎麼安排」；用「如果你想要…可以…」取代命令句。
- 不要固定開頭（避免起手式模板句）：模板是「幫你快速起稿」用，不是每篇都要長一樣。避免每段/每篇都用同一個固定起手式（例如「最省腦的起手式：…」「必做：…」「一定要：…」）；第一句要做的事只有一個：把它放進行程裡（定位/時段/適合誰）。
- 提醒集中寫一次：同一篇裡的時間帶/人潮/預約/走法/備案集中放在一處，不要每個條目重複講。
- 可決策資訊要可比較：同一類資訊用一致維度（距離/時間帶/人潮/預約/預算/替代方案），讓讀者能選而不是被帶著走。

## 真實性與來源（SSOT 寫作底線）

- 不捏造親身經驗：如果不是你真的去過/看過/聽過，就不要寫「我…」的體驗句（例如：我聽到街頭藝人唱什麼、我看到誰在旁邊做什麼）。
- 把「氛圍」寫成可支持的描述：用可觀測/可想像的語句，或加上來源語氣（例如「常見資料/口碑會提到…」「多數分享會提到…」）。
- 避免硬塞私人喜好梗：喜歡的音樂/電影/內梗等，除非是 SSOT 內容明確要走「長文/個人散文」路線，否則不要放進主內容。

### 資料不足時：用網路搜尋補缺（必做流程）

原則：寧可花時間補來源，也不要用想像補內容。 如果你手上的原始資料不夠（缺營業時間、票價、預約規則、動線、季節性、名稱拼法等），就要主動上網查核後再寫。

- 先列出缺口：你缺的是「地址/交通」「時間帶/休日」「是否需預約」「票價/艙等/份量」「限制條件（排隊/禁拍/無座位）」哪一種？缺口要寫成清單（3–10 條），後面才好逐條補齊。
- 搜尋順序（由可靠到次要）：
  - 官方：官網、官方 IG/FB、官方公告頁（通常最準，但也可能沒更新）。
  - 地圖/平台：Google Maps/Apple Maps/訂位平台（看近期評論日期，留意臨休/季節變更）。
  - 旅遊局/在地媒體/近期分享：用來補背景與常見注意事項；以 1 年內資訊優先。
- 交叉驗證：關鍵資訊（時間/票價/規則/預約）至少對到 2 個來源；如果來源互相打架，就不要寫死，必要時乾脆不要寫那條。
- 寫法：
  - 能寫得很確定的：用中性肯定句寫清楚（並避免誇飾）。
  - 只能寫到「常見說法」的：用「常見資料/多數分享會提到…」，不要裝得很確定。
  - 不夠確定就不寫：尤其是價格、交通時間、人潮強度的絕對值。

#### 圖片搜尋：用來補「氛圍描寫」的可觀測線索

目的：不是靠照片寫故事，而是把「看得到的線索」轉成 2–5 句可落地的氛圍段落。

- 去哪裡找圖（優先順序）：
  - 官方 IG/官網相簿/新聞稿（通常最能代表品牌想呈現的樣子）
  - 地圖平台的照片（比較接近一般旅人視角）
  - 訂房/訂位平台的照片（房型/座位/餐點呈現常更完整）
- 怎麼看圖（只抓可觀測欄位）：
  - 光線：自然光多不多、暖黃/冷白、白天/黃昏/夜裡的差異
  - 材質：木、石、磁磚、金屬、布料，空間是粗獷還是乾淨俐落
  - 動線：入口是敞開還是隱密、吧台/桌席比例、座位密度
  - 視野：窗景/街景/海港/山景，視線會被什麼吸走
  - 食物外觀：擺盤結構、份量尺度、常見配菜（只寫看得到的）
- 怎麼寫成段落（避免腦補）：
  - 用 1 句先定位（它是偏咖啡館/酒館/家庭料理/小吃店的哪種節奏）
  - 用 1–2 句寫空間與光線（把材質與視野寫進來）
  - 用 1 句把它接回行程（午餐休息點/宵夜收尾/雨天備案/想早回房間的那晚）
- 禁止事項：
  - 不從照片推論味道、服務態度、音量、人潮
  - 不把照片裡的人寫進敘事（穿著、國籍、情緒、對話）
  - 不把「官方宣傳照」當作人潮/排隊的證據

## 語氣與措辭（Do / Don’t）

### Do

- 用選項語氣：`如果想要更安靜的版本，可以改去 ...`
- 用可觀測的描述：`傍晚光線很漂亮，但人潮也會上來。`
- 用中性建議：`時間不夠時，可以只走 A 段，B 段留給下次。`
- 用可驗證的來源語氣：`常見資料會提到 ...`
- 每篇第一句「先定位」但不必同一句：用 1 句把它放進行程（時段/用途/適合誰），句型可以每篇都不一樣。

### Don’t

- 教訓式：`一定要先去，不然你會後悔。`
- 省力命令：`直接買 X 就好／不要浪費時間。`
- 空泛形容：`超美、超讚、必去、超推薦`（除非你立刻補一句「為什麼」）
- 未經驗證的第一人稱：`我聽到 ... / 我遇到 ... / 我一口就懂 ...`（除非真的是你的親身經驗）
- 固定起手式（boilerplate）：同一篇/同一系列文章不要反覆用同一句開頭（例如每篇都「最省腦的起手式：…」）。可以用「定位句」取代「口號句」。

## 段落組織（讓讀者讀得順）

### 建議順序（Attractions / Extensions）

1. 大圖景：這城市/地區的「主菜」是什麼（氛圍、地形、文化）。
2. 必看/經典（少量）：放最能代表城市的 3–6 個（不要堆到失焦）。
3. 選配（依興趣）：依「看展/建築/美食/街區散步/夜景」分群。
4. 小技巧：預約、人潮、最佳時間帶、走法（用短段落，不要在每個條目重複）。

### 建議順序（Stays）

1. 住哪一帶的直覺結論：用 2–4 個區域對照（安靜/熱鬧/交通/餐廳/安全感）。
2. 候選清單：每個住宿 1–2 句「定位」＋一個「注意點」。
3. 最後備註：入住時間、噪音、停車、上下樓、冷氣等（集中寫，避免散落）。

### 建議順序（Transport）

1. 一句話摘要：最適合的選擇是什麼（火車/巴士/租車），以及為什麼。
2. 步驟：訂票/進站/取票/轉乘要點（短句＋清單）。
3. 風險與備案：誤點、人潮、提早到、替代班次（中性寫法）。

## 條目模板（直接可用）

### Attractions / Extensions 條目（清單）

格式（兩種都可以，用情境挑）：

- 精簡版：`- **名稱**：一句話定位 + 一句話建議（可選）`
- 氛圍版（推薦）：`- **名稱**：一句話定位` + 下一行用短段落補「氛圍/為什麼值得」+（可選）子清單放可決策資訊
- 段落優先短文版（不想看起來像列點時用）：`- **名稱**：一句話定位` + 下面用 1–2 段把「為什麼」寫完；最後（可選）只用 1–3 個子彈列決策資訊

範例（風格示意）：

- 某個老城廣場：適合傍晚散步和吃晚餐前繞一圈；如果只想拍照，黃金時間會比正午舒服很多。
- 某個博物館：展量偏大，重點是某幾個館藏；時間不夠可以直接鎖定一層或一個主題看完就走。

範例（段落優先短文版示意）：

- 某間選品店：想帶走「這座城市的日常」時很適合把它當成購物起點。
  這類店的好不是「必買清單」，而是你繞一圈就能感覺到城市的審美：包裝、氣味、材質與陳列都在同一個節奏上。你不必每樣都買，但通常會很快知道自己最想把哪一種日常帶回家。

選品/小店/香氛（SSOT）寫法模組（合併到範例區）：

- 外觀線索（1 句）：用 1–2 個具體物件讓人想像你為什麼會停下來（例如：玻璃窗可見展覽、門口書箱、灰黑色陳列）。
- 快速定位（1–2 句）：它是什麼類型的店、主打什麼（選物/獨立出版/香氛），以及「偏觀光/偏小眾」的定位（用資料/口碑語氣）。
- 代表性選項（子清單）：列 2–4 個「最能代表它」的品項或系列；名稱/香名用 `反引號` 包起來，避免段落內粗體。
- 可決策資訊（子清單）：預算感、包裝是否適合送禮、是否需要預留時間、是否容易買完。
- 如果資料提到「店主互動/店貓/隨性營業」：用來源語氣點到即可（例如「常見分享會提到店裡有 ___」），不要寫成第一人稱故事。

範例（段落優先短文版示意：街邊小吃店）：

- 某間豬排三明治小店（Bifana）：想用一口「葡萄牙日常」快速補能量時，這種街邊小店最有效。
  Bifana 本來就是葡萄牙常見的小吃；越是到處都有，越能看出一間名店到底強在哪裡——香氣、醬汁、麵包口感與動線，通常一到現場就會有答案。若店內沒有座位也很正常，把它當作「站著吃/外帶去廣場」的行程節點就好。
  - 吃法：辣油/芥末類調味依店家提供
  - 預算：通常是低門檻小吃價位

範例（段落優先短文版示意：餐飲節點（快速補能量／午餐定食／宵夜收尾），合併版）：

- 某個餐飲節點：把餐廳寫成「你行程裡的一個功能」，而不是一串形容詞。它是快速補能量的街邊小吃？是旅行中段能坐下來喘口氣的午餐定食？還是泡完錢湯後用熱湯收尾的宵夜？定位清楚後，讀者就知道什麼時候該把它接上。
  正文只要再補 2–5 句氛圍與動線就夠：站著吃/外帶、二樓窗席、巷弄裡的黃光、或「一碗熱麵把夜晚收起來」這種節奏。最後用子清單把決策欄位收斂，讀者會更容易安排，而不是被段落淹沒。
  - 型態：立吞/外帶為主 vs 可坐下來慢慢吃
  - 時段：午餐供應時段是否固定；夜裡是否更有氣氛
  - 份量/預算：低門檻小吃 vs 定食份量剛好（是否能留肚子給甜點）
  - 限制：是否容易排隊；座位少不適合帶大行李
  - 下一步：吃完要去散步/去甜點店/回旅館休息（用一句話接回行程）

範例（段落優先短文版示意：食物介紹/料理入門＋點餐選擇題，合併版）：

- 某道/某店名物（漢堡排定食、湯咖哩等）：介紹食物時不要只說好吃；把它寫成「讀者能立刻做決定的結構」：這道菜是什麼、怎麼吃、有哪些選項、以及你會怎麼選。
  先用一段把「料理的差異」講清楚（例如濃稠醬汁 vs 香料湯高湯；或鐵板上桌的漢堡排定食），再用一段把「點餐的選擇題」收斂（醬汁/份量/組合款），最後才補上 1–2 句畫面（滋滋作響、半熟蛋、蔬菜色彩）讓它有記憶點。
  - 主軸：名物是什麼（漢堡排/湯咖哩等）
  - 吃法：飯怎麼配、湯/醬怎麼入口；是否常見分開上桌
  - 選項：醬汁（黑/白等）/份量（150g/300g 等）/組合款（炸蝦/可樂餅等）
  - 配角：米飯本身是否是賣點（產地/玄米/可否續）
  - 候位：小店座位少或名店熱門時，午餐要預留等待時間

範例（段落優先短文版示意：奇遇型古物店）：

- 某間古物店：如果你想把旅程裡的「偶遇」留一點位置，古物店很適合當作半天的意外支線。
  好的古物店不靠「必買清單」，而是靠空間與人的氣質：老建築、滿到像迷宮的物件陳列、以及店主/店貓這些細節，常會讓你覺得自己走進了一個小小場域。這類店的營業與動線也可能更隨性，建議把它排在「時間有彈性」的那段行程裡。
  - 逛法：用掃視找自己會反覆回頭的那一區，再慢慢收斂
  - 提醒：營業時間/可否拍照/付款方式

### Stays 條目（清單）

格式：`- **住宿/區域**：一句話定位 + 一個注意點`

範例（風格示意）：

- 某區（交通樞紐）：移動最方便，晚上也比較有店；缺點是車站周邊可能比較吵，挑房型要注意隔音。

範例（段落優先短文版示意：住宿）：

範例（段落優先短文版示意：住宿類型（設計旅館／度假村／品牌分館），合併版）：

- 某間住宿：把住宿寫成「你想要的夜晚版本」，而不是房間規格表。它是讓你早點回房間、把夜晚過得像一段展覽的設計旅館？是把時間拉慢、幾乎不出門的度假村？還是同一個品牌在不同城市的分館，讓你用熟悉的語言去認識新的街區？
  正文只要再補 2–5 句就夠：公共空間像藝廊/策展、窗景把夕陽或海港拉進來、房間用挑高/分區把小空間變好用、或園區裡的水與風讓你願意發呆。最後用子清單把決策欄位收斂，讀者會更容易選，也更容易安排。
  - 交通：離車站遠不遠；對外移動更像散步還是更適合叫車
  - 定位：設計旅館（想早回房間）/度假村（想不出門）/品牌分館（想對照）
  - 房型：是否有特色房（例如藝術家合作房、樓中樓/榻榻米分區）
  - 加分項：早餐、在地導覽、園區活動（把它寫成「什麼時候用得到」）
  - 注意點：隔音、清潔、行李動線、周邊補給

### Transport 段落（步驟清單）

範例（風格示意）：

- 買票：線上先看班次與票價，若不確定時間就選可改票的票種。
- 到站：提早 20–30 分鐘到，尤其是行李多或要找月台時。
- 備案：如果錯過班次，先確認是否可改票；不行就直接改搭下一班／改巴士（通常班次更多）。

範例（段落優先短文版示意：跨城列車/高鐵）：

- 跨城列車（高鐵/私鐵）：如果你會在國內跨城市移動，火車常是「舒適＋省心」的選項；關鍵差別通常不在車速，而在你買的票「能不能改／能不能退」。
  介紹這類交通時，先用 2–4 個名詞把規則講清楚就好（艙等、票價型、是否含選位/候車服務），再用一句話把它放回旅途焦慮：行李、月台資訊什麼時候公布、以及你怎麼把混亂降到最低。
  - 訂票：越早通常越便宜
  - 票種：`Economy` 類通常較便宜但彈性差；`Flex` 類較貴但可改/可退
  - 安心感：如果有大行李，可留意是否有「出入口較單純/需驗票進入」的車廂或候車區
  - 現場節奏：月台/站台資訊可能接近發車才公布，提早到站並保留上車緩衝會比較穩

### Itinerary（行程文章／一日攻略）

寫法：1 段把「這是什麼樣的一天」講清楚 → 1 段把「節奏」講清楚 → 子清單只放可決策資訊（交通/預約/時間帶/替代方案）。正文不需要每段都變成「幾點幾分」，重點是讓人看完能照抄節奏。

子型：低摩擦/帶長輩版本（同樣是一日安排的骨架）：

- 動機：為誰而做（父母/長輩嚮往 ___，所以安排 ___）。
- 取捨：用預算換舒適（行李、轉乘、體力摩擦降到最低）。
- 骨架：交通／住宿／行程／季節（讓讀者好照抄）。
- 主方案 + 替代方案：同時寫出代價（時間更長/需要轉乘/要拖行李）。
- 數字必備：車程、費用、可坐幾人、每站停留多久（讓讀者能比較）。
- 住宿亮點要具體：視野、餐食、動線、浴場形式、步行補給點。

範例（段落優先短文版示意：村落一日/主題一日）：

- 某個村落一日（陶器＋在地早餐＋海景咖啡）：如果你想認識一個地方的「日常感」，一日行程不必塞滿景點，反而適合抓一個主題把它走深。
  這種行程的節奏很簡單：早上用一間很在地的早餐店把人群與生活感打開；上午安排一個「只有這裡才有」的體驗（導覽/工坊/手作）；中午用一碗代表性在地菜把味覺收束；下午再留給海邊/高處/咖啡店，把時間慢慢放回來。
  - 車程：距離大城市約 1 小時等級
  - 需要預約：若有導覽/體驗，通常需要先看報名方式
  - 雨天備案：把下午段改成工坊/室內店舖；或縮成半日只走「早餐＋體驗＋午餐」

## 讓內容更「有魅力」但不變長的技巧

- 加 1 句文化/情境就夠：例如「這條街的特色是…」「這一帶的節奏是…」。
- 把資訊變成比較：不要堆資訊點；把同類資訊放在一起對照（例如兩個街區的差異）。
- 刪掉重複的提醒：同一篇裡的「人潮/預約/時間帶」集中寫一次即可。

## 長文/個人經驗（另檔）

若要寫「生活切片、散文、朝聖、過境一日、活動日誌、旅遊誌長文」等長文技巧，集中放在 `docs/writing-style-longform.md`。

## 文章風格參考：公共場域（廣場／市集／市場／公共設施，融合版）

這一組的共通點是「把場域寫成可安排的節奏」：你先幫讀者定位它在城市裡的角色，再把動線與時間帶講清楚。子型的差異只在「你要解除的焦慮」不同（點餐、觀光濃度、時間差、動線與規則）。

### 共用骨架（先看這段就夠）

- 立題（1 段）：一句把它放進城市節奏裡（夜晚/週末/雨天/行程空檔）。
- 定位（1 句）：它更像生活場域、觀光入口，還是公共設施（用一句話讓人不會走錯期待）。
- 動線（3 句）：先繞一圈 → 找強項/看規則 → 再下手（避免寫成命令）。
- 時間帶（1 句）：什麼時候最好、什麼時候比較擠（用對比講清楚）。
- 收束（1 句）：用一句話回答「它適合誰／為什麼值得排」，並接回下一步（去吃飯/去散步/回旅館）。

範例（共用骨架的氛圍段落示意）：

- 某個老城廣場／市場口：傍晚很適合來這裡慢慢走一圈。
  光線落下來時，石材和立面會變得很柔和；你不一定要做什麼，站著看人群流動就很有旅行感。
  - 時間帶：黃金時間最舒服；正午比較晒
  - 人潮：晚餐前後會上來

### 子型 A：夜市/美食場域（重點是解除「點餐焦慮」）

- 友善性寫清楚：有英文/中文菜單更好；沒有就看圖片/看實品；最後用「指食材比手畫腳」收束。
- 亮點寫法：挑一個「結構差異」寫深（例如餐飲區集中、半開放鐵皮屋、座位區很長、吆喝聲與香氣）。

### 子型 B：觀光市場/觀光場域（重點是「短歷史 + 平衡評價」）

- 短歷史三段式（1 段）：成立 → 轉型 → 變成今天的樣子（不要百科）。
- 限制條件：獨旅/食量小/不能分食，會讓「我怎麼選」合理。
- 平衡評價：承認可能較貴/較觀光，但它是初訪的味覺入口。
- 尾韻：留一個小遺憾（忘了買某樣）很有效，會自然帶出「下次」。

### 子型 C：跳蚤市場/挖寶場域（重點是「時間差 + 挖寶哲學」）

- 時間差：週二 vs 週六（人潮/攤位/周邊店家）先講清楚。
- 舞台：年份/廣場/地標/視野，用 2–3 個點讓讀者知道自己在哪裡。
- 挖寶哲學：價值主觀、允許空手而回（不要硬推消費）。

### 子型 D：澡堂/公共設施（重點是「定位 + 規則 + 下一步」）

- 線索開場：路過的招牌/看板/廣告（像「行程有餘裕時的起心動念」）。
- 背景 1–2 句：發現/開業/整修（不要百科）。
- 先定位：地方澡堂 vs 觀光設施（票價 vs 精緻/動線/空間）。
- 泉質/觸感：用 1–2 個可感受的詞（顏色、包住的暖意），避免堆術語。
- 時間帶：白天/入夜/黃昏各有不同。
- 收束：泡完的下一步（飲料/晚餐/回程），把行程收得完整，但不寫成命令。

### 套回本 repo：適合放哪裡？

- 市集/市場：`attractions.*.md` 的「美食/市場」小節；或城市頁的「夜晚」玩法段落。
- 澡堂/公共設施：`attractions.*.md` 的「散步/海邊/黃昏」加分點；或 `stays.*.md` 的「泡湯/桑拿」段落。


## 逐檔完稿自檢（每篇改完必做）

- 短文先段落：正文先用 1–2 段講清楚「這是什麼／為什麼值得」；清單只放決策欄位。
- 可掃描：每個條目的第一句都能在 3–5 秒內掃到「這是什麼」。
- 可安排：至少有一處寫出時間帶/動線/下一步（接回行程）。
- 可決策：至少提供 1 個替代方案或「時間不夠的版本」。
- 不捏造：SSOT 不寫未驗證的第一人稱；資料不夠就先去查再寫（照上面的「資料不足時」流程）。
- 不靠粗體：段落內不使用粗體畫重點（必要時改寫句子結構）。
- 氛圍要可觀測：氛圍句要落在光線/材質/動線/視野等線索上；需要時用「圖片搜尋」補齊觀察點。
- 不寫保險括號：不要用「以…」這類語氣替資訊買保險；要嘛查到再寫，要嘛乾脆不寫那條。

## 葡語短句與「空耳」寫法（讓人知道自己在點什麼）

原則：葡語用 `反引號` 包起來，後面用括號給中文意思與空耳（空耳是近似發音，目的是讓人敢開口，不追求口音完美）。

建議格式：

- `` `Um galão, por favor.`（一杯大杯奶咖；空耳：嗯 嘎朗，普法沃） ``
- `` `Uma meia de leite, por favor.`（一杯瓷杯半奶咖；空耳：烏瑪 梅亞 德 雷特，普法沃） ``
- `` `Um café / uma bica, por favor.`（一杯濃縮；空耳：嗯 咖啡／烏瑪 比卡，普法沃） ``

如果需要加條件，照同一個節奏往後疊：

- `` `… com espuma / sem espuma / pouca espuma`（要奶泡／不要奶泡／奶泡少一點；空耳：空 伊斯普瑪／桑 伊斯普瑪／波卡 伊斯普瑪） ``

## UI 友善的 authoring 注意事項（避免 silent wrong / missing）

- UI 目前支援的「可用寫法」（最常用的 Markdown 子集）：以下這些會在 App UI 裡正常顯示（來源：`docs/src/content/README.md`）。
  - `**粗體**`
  - `` `行內 code` ``
  - 連結：`[文字](https://example.com)`（以及直接寫 `https://...`）
  - 清單：`- item`、`1. item`
  - 勾選清單：`- [ ] item`、`- [x] item`
  - 引言：`> quoted text`
- UI 內容生成方式（避免誤會）：App 不會在 runtime 直接渲染整份 Markdown；會先把 `docs/src/content/*.md` 轉成 `docs/src/generated/*.ts` 的資料，再由前端 UI render（細節見 `docs/src/content/README.md`）。
- 自訂語法（目前在內容中有使用）：
  - `@card: ...`：用於段落/小節的「一行摘要」卡片（例如 `attractions.*.md` 常見）。
  - `{{gallery:Title|url|url|...}}`：圖片 gallery token（必須整行）。
- 維持 block 分隔：段落、清單、標題之間留空行（UI 解析是依空行切 block）。
- 不要把 `{{gallery:...}}` 拆行：token 必須完整在同一行。
- 避免複雜巢狀語法：不要在同一段落裡混用太多層的粗體/斜體/反引號；需要強調就改句子結構。
- 標題不要加粗：`##/###/####` 自己已經是視覺結構。
